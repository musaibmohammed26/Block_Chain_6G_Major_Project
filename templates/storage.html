Hybrid Storage Policy
ON-CHAINBlockchain Component

Stores metadata, transaction proofs, and critical state information directly on the blockchain. Limited to small data sizes (<500 bytes) for optimal blockchain performance.

  • Immutable storage
  • Full consensus verification
  • Indexable on blockchain
  • Higher verification cost
OFF-CHAINDistributed Component

Stores large data volumes in distributed systems (IPFS, cloud storage). Referenced from blockchain via content hash. Optimized for scalability and cost efficiency.

  • Cost-efficient storage
  • High scalability
  • Content-addressable (IPFS)
  • Lower retrieval latency
